## 📌 Executive Summary

현재 News Spectrum의 UX가 **"기능 전시형"**으로 너무 복잡함.
**X.com(트위터) + 뉴닉** 스타일의 가볍고 직관적인 피드로 전환 필요.

**핵심 철학:**
> "일반적인 커뮤니티형 피드에 우리 서비스를 조미료처럼 자연스럽게 녹여내서,
> 레이어가 많음에도 간결히 이미지로 제공하고,
> 일반 기사는 심심해 보이게 만들기"

---

## 🔴 현재 문제점

### 1. 직관성 부족
- 2-column grid → 정적이고 데스크톱 중심
- MegatopicCard가 너무 복잡 (Country Breakdown 펼치기, 왜 중요한가 등)
- 사용자가 "대단한 방식으로 알려드려요!" 느낌을 받음

### 2. 과도한 정보 노출
**현재 카드 구조:**
```
MegatopicCard (300-400px 높이)
├─ 제목 (한글/영문)
├─ 메타 정보 (날짜, 기사 수)
├─ "왜 중요한가" 섹션 (80자 요약)
├─ Global Stance (스펙트럼 바)
├─ Country Breakdown 버튼
│   └─ 펼치면 → 모든 국가 스펙트럼 표시
└─ [View Full Analysis] 버튼
```
→ **너무 많은 레이어, 사용자 압도**

### 3. 사용성 문제
- 피드 스크롤이 무거움
- 한 번에 보이는 토픽 수 적음 (2개만)
- 클릭 전에 너무 많은 정보 노출
- 모바일에서 스크롤 피로도 높음

## 🎯 목표 UX

### 레퍼런스 분석

**X.com (트위터):**
- Single column feed
- 빠른 세로 스크롤
- 간결한 프리뷰 (제목, 이미지, 메타)
- 탭하면 상세 내용

**뉴닉:**
- 친근한 톤 ("~해요", "~군요")
- 이모지 활용 (💡, 🌍, 📰)
- 쉬운 문법
- 간결한 요약

**우리가 원하는 위치:**
```
뉴닉 ←───────── [여기!] ───────→ X.com
(친근한 톤)              (가벼운 사용성)
```

### 새로운 사용 흐름

```
1. 피드 스크롤 (가볍게)
   ↓
2. 관심 토픽 발견
   ↓
3. 한 번 탭/클릭
   ↓
4. 상세 서랍 열림 (모든 정보)
   ↓
5. 닫고 다시 피드로
```

## 📊 예상 효과

### 긍정적 효과
1. **사용성 대폭 개선**
   - 피드 스크롤 속도 2-3배 향상
   - 한 화면에 3-4개 토픽 → 8-10개 토픽

2. **진입 장벽 낮춤**
   - 첫 인상: 간결하고 깔끔
   - 클릭 전: 부담 없음
   - 클릭 후: 풍부한 정보

3. **모바일 UX 최적화**
   - 세로 스크롤 최적화
   - 한 손 사용 가능
   - 빠른 소비 가능

### 잠재적 리스크
1. **정보 손실 우려**
   - 해결: Drawer에 모든 정보 유지

2. **클릭 필요 (추가 액션)**
   - 해결: 피드에서도 핵심 정보는 보임 (스펙트럼)

3. **구현 시간**
   - Phase 1만: 2-3시간
   - Phase 1+2: 5-6시간
