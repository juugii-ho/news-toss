# Product Roadmap - News Toss (News Spectrum)

> 이 문서는 뉴스토스 MVP 이후의 주요 기능 개선 및 확장 계획을 정의합니다.
> 이 로드맵은 S(PO)의 결정에 따라 `PRIORITY.md`와 `WORK.md`의 구체적인 태스크로 전환될 수 있습니다.

---

## 🚀 Post-MVP: Next Steps

### 🌟 1. 핵심 기능 개선 (Core Feature Enhancements)

#### 1.1. Local 탭: '트렌드 스코어' 도입
- **목표:** 단순 기사 수(`article_count`)가 아닌, 시간 요소를 반영한 '트렌드 스코어'를 도입하여 "지금 가장 뜨거운" 토픽을 더 역동적으로 보여줍니다.
- **관련 제안:** `DT-003`

#### 1.2. Global 탭: 국가 확장 및 비교 기능
- **목표:** 현재 5~10개국 중심의 분석을 더 많은 국가로 확장하고, 사용자가 직접 2~3개 국가를 선택하여 입장을 비교해볼 수 있는 기능을 제공합니다.

#### 1.3. UI/UX 고도화
- **데스크톱 전용 UI/UX:** 현재의 모바일 뷰 중앙 정렬 방식에서 벗어나, 넓은 화면을 활용하는 데스크톱 전용 레이아웃을 설계합니다.
- **다크 모드:** 사용자의 시력 보호와 선호도에 맞춰 다크 모드 테마를 지원합니다.

---

### ✨ 2. 신규 기능 (New Features)

#### 2.1. 사용자 계정 및 개인화
- **목표:** 사용자 회원가입을 도입하고, 관심 국가, 선호 카테고리, 구독한 토픽 등을 기반으로 개인화된 뉴스 피드를 제공합니다.
- **기대 효과:** 사용자 리텐션 및 충성도 증대.

#### 2.2. 어드민 대시보드
- **목표:** 내부 운영을 위한 어드민 페이지를 개발합니다.
- **주요 기능:**
    - 뉴스 소스 관리 (`is_active` 토글, 신규 추가)
    - 토픽 수동 병합/편집
    - 스탠스 및 카테고리 수정
    - 주요 통계 대시보드
- **관련 제안:** C의 `TEAM_REQUESTS_AND_PROPOSALS.md`

#### 2.3. 소셜 기능
- **목표:** 사용자가 인사이트를 공유하고 토론할 수 있는 환경을 제공합니다.
- **주요 기능:**
    - 'VS 카드' 또는 'Gravity Bowl' 화면을 이미지로 공유하는 기능
    - 각 토픽에 대한 간단한 댓글 또는 투표 기능

#### 2.4. 유료 구독 모델
- **목표:** 심층 분석 리포트, 광고 제거, 고급 개인화 기능 등을 제공하는 유료 구독 모델을 도입하여 수익화를 모색합니다.

---

### 🛠️ 3. 데이터 파이프라인 및 백엔드 고도화 (Technical Improvements)

#### 3.1. '스마트 피드' 수집 시스템
- **목표:** RSS 피드별 `last_collected_at`을 기록하여 불필요한 중복 수집을 방지하고, 반복적으로 실패하는 피드를 자동으로 비활성화하여 파이프라인의 효율성과 안정성을 높입니다.
- **관련 제안:** `TASK-002`

#### 3.2. 데이터 품질 검증 파이프라인
- **목표:** 수집된 데이터(기사, 요약 등)의 품질을 자동으로 검증하고, 일정 기준 미달 시 다음 단계로 넘어가지 않도록 하여 데이터의 신뢰도를 높입니다.
- **관련 제안:** C의 `TEAM_REQUESTS_AND_PROPOSALS.md`

#### 3.3. 파이프라인 모니터링 시스템
- **목표:** 데이터 파이프라인의 각 단계(수집, 번역, 분석 등)별 성공/실패 수, 처리 시간 등의 메트릭을 수집하고 시각화하여 운영 상태를 한눈에 파악합니다.
- **관련 제안:** C의 `TEAM_REQUESTS_AND_PROPOSALS.md`

#### 3.4. 고도화된 캐싱 전략
- **목표:** `Next.js`의 기본 캐싱 외에 `Redis`와 같은 외부 캐시 저장소를 도입하여, 더 빠르고 안정적인 API 응답 속도를 보장합니다.
- **관련 제안:** C의 `TEAM_REQUESTS_AND_PROPOSALS.md`
